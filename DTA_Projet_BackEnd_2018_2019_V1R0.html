DTA Projet Back-End 2018-2019

Objectifs

L’objectif de cette semaine est de réaliser un projet de développement back-end (PHP - MySQL).

Cela doit permettre à chaque élève de progresser sur certains savoir-faire et savoir-être fondamentaux du développement web :

  • Développement back-end
    • Développement PHP
    • Conception base de données relationnelles.
  • Travail collaboratif
    • Découpage du projet en tâches élémentaires
    • Sprints
  • Présentation du résultat

L’objectif n’est pas forcément d’aboutir à un site absolument esthétique (nous vous recommandons d’utiliser Bootstrap pour que le rendu reste acceptable).
Par contre l’accent doit être mis sur les validations et les traitements de formulaires, les recherches et les mises à jour en base de données.

Planning

Le projet se déroule semaine 02, du 07 au 11 janvier.
La présentation du projet a lieu lundi 07 janvier en 1ere partie de matinée.
Les équipes et les sujets doivent être choisis lundi 07 janvier en fin de matinée au plus tard.
Les soutenances ont lieu vendredi 11 janvier.

Modalités

Le projet se fait par équipe de 2 élèves (2 équipes de 3, 18 équipes de 2).

Le choix des équipes vous est laissé libre, mais vous ne devez pas vous remettre avec votre partenaire du projet front-end.

Les élèves sont en autonomie, avec présence obligatoire (feuille de présence à signer comme d’habitude) dès le choix des équipes et des sujets (lundi matin), jusqu’au jeudi soir.
Le vendredi, par 1/2 groupe, les équipes présenteront leurs réalisations.
Un temps de 30 minutes (questions comprises) est prévu par présentation.
La présentation doit obligatoirement aborder les points suivants :

  • Présentation générale du modèle de base de données
  • Choix techniques généraux effectués
  • Méthodologie de travail à deux
  • Focus sur 2 points spécifiques du code PHP (1 pour chaque élève)

Les ordres de présentation seront communiqués dans la semaine.

Sujets

Application de carnet de prospects

L’application permet de gérer des prospects :

  • Liste de prospects (avec possibilité de filtre)
  • Visualisation / Ajout / Suppression / Modification d’un prospect
  • Connexion / Inscription / Visualisation / Ajout / Suppression / Modification de ma fiche utilisateur
  • Champs nom, prénom, date de naissance, mail, téléphone pour les prospects…
  • Champs nom, prénom, email, password pour les utilisateurs…
  • Gestion de commentaires pour les prospects

Application e-commerce

L’application permet de présenter un catalogue de produits (le type de produit est laissé libre : BD, chaussures…) :

  • Liste de produits (avec possibilité de filtre), regroupées par catégorie
  • Visualisation d’un produit
  • Connexion / Inscription / Visualisation / Ajout / Suppression / Modification de ma fiche utilisateur
  • Champs label, description, photos, commentaires pour les produits…
  • Champs nom, prénom, email, password pour les utilisateurs…
  • Gestion d’un panier utilisateur

Application de TODO list

L’application permet de gérer des TODO lists :

  • Liste d’actions (avec possibilité de filtre), regroupées par statut
  • Visualisation / Ajout / Suppression / Modification d’une action
  • Connexion / Inscription / Visualisation / Ajout / Suppression / Modification de ma fiche utilisateur
  • Champs titre, description, statut, responsable pour les actions…
  • Champs nom, prénom, email, password pour les utilisateurs…
  • Gestion de commentaires pour les actions

Application météo

L’application propose de stocker des relevés météo pour des lieux donnés :

  • Liste de lieux (avec possibilité de filtre)
  • Visualisation / Ajout / Suppression / Modification d’un lieu
  • Connexion / Inscription / Visualisation / Ajout / Suppression / Modification de ma fiche utilisateur
  • Champs label, description, latitude, longitude pour les lieux…
  • Champs nom, prénom, email, password pour les utilisateurs…
  • Gestion (création, visualisation, suppression…) de relevés météo (températures, conditions, humidité…) pour les lieux
